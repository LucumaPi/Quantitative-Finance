3 // Test of set.Concerned with interactions between sets. In particular, we look
4 // at some set-like operations:
5 //
6 // set_union
7 // includes (is A a subset of B?)
8 // set_intersection
9 // set_difference
10 // set_symmetric_difference (see section 20.28 in Musser)
11 //
12 // These algorithms are in <algorithm> and they use the assumption that all elements are
13 // sorted using the operator <. Another variant allows us to use a comparitor function.
14 //
15 // We use an insert iterator to compute these set functions (thanks to JD).
16 //
17 // Finally, a number of generic algorithms are defined that promotes Information Hiding
18 // so that STL is a bit more user-friendly. In particular, we do not have to worry about
19 // iterating or using insert iterators in client code. This approach would form the
20 // beginnings of a Microkernel pattern for STL (see the patterns book by Buschmann).
21 //
